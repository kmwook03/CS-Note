## Deamon Process
시스템 부팅 때 실행 시작하여 시스템 셧다운 시 종료되는 프로세스다.
백그라운드에서 실행되고 controlling terminal을 가지지 않는다.

### Coding Rule
#### 1. fork() 호출 후 부모 프로세스 exit
부모 프로세스를 죽이지 않으면 터미널 프롬프트를 반환 받을 수 없다.
즉, 터미널 프롬프트를 반환 받고 자식 프로세스를 고아로 만들어 백그라운드에서 돌리기 위해서는 부모를 죽여야한다.

#### 2. setsid() 호출로 새로운 세션 생성
프로세스가 원래 속해있던 셸의 터미널 제어로부터 벗어나기 위해서는 새로운 세션으로 독립해야한다.
setsid() 함수는 현재 프로세스를 새로운 Session Leader로 만들고, Process Group Leader로 만들고, 기존의 Controlling Terminal과 연결을 끊는다.

> **fork() 후 자식에서 setsid()를 호출하는 이유**
 setsid()는 Process Group Leader 프로세스에서 호출할 수 없기 때문이다. fork()를 통해 생성된 자식 프로세스는 GID는 부모에게서 상속받지만 PID는 새로 할당되기 때문에 PID와 GID가 달라 수학적으로 Process Group Leader가 아님이 보장된다.

#### 3. 현재 프로세스의 작업 디렉토리를 루트 디렉토리(`/`)로 변경
자식 프로세스는 부모 프로세스의 작업 디렉토리를 그대로 상속받는다. 만약 데몬을 실행한 위치가 특정 마운트된 파일 시스템이라면 데몬은 시스템 종료 까지 해당 디렉토리를 점유하게 된다.
이 때 시스템 관리자가 해당 파일 시스템을 umount하려고 하면, 데몬에 의해 실패하게 된다.
따라서 이를 방지하고자 chdir("/")를 호출하여 데몬의 작업 공간을 절대 마운트 해제되지 않는 루트 디렉토리로 옮긴다.

#### 4. 파일 생성 권한 마스크를 0으로 초기화
데몬 프로세스가 동작 중 로그 파일이나 설정 파일 등 새로운 파일을 생성할 때 부모에게 상속받은 마스크 때문에 의도했던 파일 권한이 거부될 수 있다.
따라서 `umask(0)`를 통해 마스크를 초기화한다.
