## File Organization

**파일(File)** 의 논리적 구성은 레코드(record)들의 시퀀스(sequence)다.

파일은 운영체제에서 기본 구성 요소로 제공되기 때문에 파일을 논하기 전 **파일 시스템(File System)** 의 존재를 가정해야 한다.
즉, 논리적 데이터 모델을 파일로 표현하는 방법에 대해 고려해야 한다.

각 파일은 **블록(block)** 이라고 불리는 고정 길이 저장 단위로 논리적으로 나눠져있다.
블록은 몇 개의 레코드를 포함하며 우리는 블록 크기보다 큰 레코드는 없다고 가정한다.
또한 각 레코드는 하나의 블록에 완전히 포함되어야 한다는 요구사항도 추가한다. 즉, 어떤 레코드도 서로 다른 블록에 찢어져서 저장되어서는 안된다.

### Fixed-Length Records

"JOJO"라는 파일이 다음과 같이 정의된다고 가정하자.

```
typedef JOJO = record
                NAME varchar(20);
                STAND varchar(20);
                age numeric(8, 2);
               end
```

문자는 1byte, numeric(8,2)는 8bytes를 차지한다고 하면 JOJO 파일의 크기는 48bytes다. (각 길이가 고정이기 때문)

이렇게 파일을 저장하면 쓰고 읽는 것은 간단하다. 첫 레코드에 48 bytes를 쓰고 다음 48 bytes는 두 번째 레코드에 쓰는 것이다.

그러나 이 방식은 문제가 있다.

1. 블록 크기가 48bytes의 배수가 아니라면 일부 레코드는 블록 경계를 넘길 수 있다.

2. 레코드 삭제가 어렵다.
